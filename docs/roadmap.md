# 开发路线图

## Phase 0：慢请求可观测性（1–2 周）

- Trace ID 和请求 Attempt 模型；
- 全链路分段耗时；
- 参数摘要和脱敏审计；
- 实际接口、模型、渠道和出口记录；
- 慢请求告警；
- Trace 详情页；
- 不修改流量的影子部署。

验收：任意慢请求可以判断耗时发生在哪个阶段。

## Phase 1：透明代理与规则引擎（2–3 周）

- JSON、form、multipart；
- 参数过滤、替换、拒绝和模型映射；
- 规则校验、版本、影子、灰度和回滚；
- 无规则快速通道；
- 普通响应和 SSE。

## Phase 2：图片处理与 OSS（2–3 周）

- 图片探测、像素限制和参考图压缩；
- 有界 worker pool；
- Base64/Data URL/临时 URL 转 OSS；
- SSRF 防护；
- OSS 故障降级。

## Phase 3：网络出口与渠道分组（2–3 周）

- 国内外出口；
- 健康检查和指标；
- 熔断与半开；
- 渠道组、能力标签和回退链；
- 并发限制和安全重试。

## Phase 4：异步任务（3–4 周）

- 统一任务状态机；
- 上游同步/异步适配；
- 轮询、webhook 和任务恢复；
- 对外异步接口；
- OSS 结果持久化；
- 幂等和取消。

## Phase 5：复杂计费（2–3 周）

- 估价、预留和最终结算；
- 分辨率、质量、数量及渠道计价；
- 表达式和规则快照；
- 退款、补扣、对账和明细。

## Phase 6：生产加固（约 2 周）

- 压测和性能回归；
- 故障注入；
- 安全审计；
- 数据保留和清理；
- 多实例、灰度、容灾和回滚手册。

## 里程碑原则

- Phase 0 独立交付，不等待其他功能。
- 每个阶段必须同时交付测试，不设“以后补测试”的任务。
- 新协议适配器没有契约测试不得合并。
- 新计费路径没有准确金额和幂等测试不得启用。
- 新图片处理策略没有峰值内存测试不得进入生产。
