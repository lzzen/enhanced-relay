# 待评审决策

以下问题应在编码前或对应阶段开始前形成 ADR。

## P0：必须先确定

- [ ] 增强网关与现有系统谁是余额和账本的最终权威？
- [ ] 现有系统能否稳定透传 `traceparent` 或其他关联 ID？
- [ ] Egress 如何标识 Provider，且不允许客户端伪造目标？
- [ ] 客户端 → Ingress → 现有系统 → Egress 是否存在代理循环？
- [ ] 生图创建请求发生连接中断时，如何判断是否允许重试？
- [ ] OSS 失败默认返回 Base64、上游 URL，还是整体失败？
- [ ] 调试快照由谁授权，最长保存多久？

## P1：实现前确定

- [ ] 规则表达语言自研 DSL、JSON Rule，还是嵌入现有表达式引擎？
- [ ] multipart 修改后如何保证字段顺序、文件名和 Content-Type 兼容？
- [ ] 图片压缩的默认格式、质量、最大长边和最大像素是多少？
- [ ] 异步任务使用 DB 任务表、Redis Streams、NATS 还是其他队列？
- [ ] 海外出口如何部署和安全接入？
- [ ] Provider Adapter 的稳定接口和能力声明是什么？

## P2：规模化前确定

- [ ] 日志使用 PostgreSQL、Loki 还是 ClickHouse？
- [ ] 图片处理是否迁移到 libvips 独立服务？
- [ ] 控制平面和数据平面是否独立扩缩容？
- [ ] 多地域任务和配置如何保持一致？

## 评审者验证清单

- [ ] 能否用一条 Trace 解释 800 秒请求的每一段耗时？
- [ ] 日志系统完全不可用时，业务是否仍可运行？
- [ ] 100 个大图并发是否会耗尽内存？
- [ ] 客户端断开后，所有可取消工作是否终止？
- [ ] 上游已受理但响应丢失时，是否可能重复生图或重复扣费？
- [ ] 任意历史费用能否用冻结规则完整重算？
- [ ] 规则错误是否可以立即回滚到透明代理？
- [ ] 所有外部 URL 下载是否阻止 SSRF 和重定向绕过？
- [ ] 二进制和 Docker 两种部署是否运行同一套自动化契约测试？

## AI 自动验收门禁清单

- [ ] 每条 P0/P1 需求是否绑定通过的验收用例（可追溯矩阵全绿）？
- [ ] 核心包变异测试分数是否达阈值（证明测试真的能抓 bug）？
- [ ] Golden 变更是否均有显式标注与审核？
- [ ] 测试断言是否存在未说明的“变松/跳过”？
- [ ] 性能、峰值内存、goroutine 泄漏是否为会让 CI 变红的自动断言？
- [ ] `make verify` 是否在干净机器上可无人值守、可重复运行并产出机器可读证据？

（详见 `docs/ai-testing-acceptance.md`）
